# Keeper — Книга Учёта Доходов и Расходов

##1. Project Overview

**Название:** Keeper
**Тип:** TUI-приложение (Text User Interface)
**Функционал:** Учёт доходов и расходов бизнеса с сохранением данных в Excel-файл
**Целевая аудитория:** Предприниматели, бухгалтера малого бизнеса

##2. UI/UX Specification

### Layout Structure

- **Режим отображения:** Одна панель с списком транзакций (подобно vim-сплитам)
- **Навигация:** Vim-style (h/j/k/l или стрелки)
- **Модальные окна:** Для добавления/редактирования транзакций, настроек

### Visual Design

**Цветовая палитра (Dark Theme — Dracula-inspired):**
- Background: `#1e1e2e` (глубокий тёмно-синий)
- Surface: `#313244` (карточки, панели)
- Primary: `#89b4fa` (акцент, кнопки)
- Secondary: `#a6e3a1` (доходы — зелёный)
- Tertiary: `#f38ba8` (расходы — красный)
- Text Primary: `#cdd6f4`
- Text Secondary: `#a6adc8`
- Border: `#45475a`

**Типографика:**
- Шрифт: Monospace (для TUI)
- Заголовки: Bold,18px
- Тело: Regular,14px
- Даты/суммы: Semi-bold,14px

**Отступы:**
- Padding карточек:12px
- Gap между элементами:8px
- Border radius:8px

### Components

1. **Header Bar**
 - Название приложения "KEEPER" (слева)
 - Текущий файл (справа)
 - Состояние: обычное / режим редактирования

2. **Transaction List**
 - Список транзакций с датой, названием, суммой
 - Доходы — зелёная метка "IN"
 - Расходы — красная метка "OUT"
 - Выбранная строка подсвечена (background: primary с low opacity)

3. **Status Bar (Footer)**
 - Клавиши быстрого доступа (vim-style)
 - Общая сумма доходов/расходов за текущий период

4. **Modal: Add Transaction**
 - Поля: Дата (date picker), Название (input), Сумма (input), Тип (IN/OUT toggle)
 - Кнопки: Сохранить (Enter), Отмена (Esc)

5. **Modal: Settings**
 - Путь к файлу
 - Имя листа Excel

### Keyboard Shortcuts (Vim-style)

- `a` — добавить транзакцию
- `e` — редактировать выбранную транзакцию
- `d` — удалить транзакцию
- `s` — создать новый файл (split)
- `o` — открыть существующий файл
- `Ctrl+q` / `q` — выход
- `j/k` или `↓/↑` — навигация по списку
- `Enter` — подтверждение в модале
- `Esc` — отмена / закрытие модала

##3. Functionality Specification

### Core Features

1. **Просмотр транзакций**
 - Загрузка данных из Excel-файла
 - Отображение в виде списка
 - Сортировка по дате (новые сверху)

2. **Добавление транзакции**
 - Дата: по умолчанию сегодня, можно выбрать
 - Название: текстовое поле (до100 символов)
 - Сумма: число с двумя знаками после запятой
 - Тип: Доход (IN) / Расход (OUT)

3. **Редактирование транзакции**
 - Изменение любого поля
 - Сохранение в Excel

4. **Удаление транзакции**
 - Подтверждение перед удалением
 - Удаление из Excel

5. **Создание нового файла (Split)**
 - Текущий файл остаётся без изменений
 - Создаётся новый Excel-файл
 - Пользователь выбирает имя/путь нового файла
 - Первый лист: "Транзакции"

6. **Открытие файла**
 - Выбор существующего Excel-файла
 - Чтение данных

### Excel Structure

```
Файл: keeper_2024_01.xlsx
Лист: Транзакции
Колонки:
 A - Дата (YYYY-MM-DD)
 B - Название
 C - Сумма
 D - Тип (IN/OUT)
```

### Edge Cases

- Пустой файл — создать новый с заголовками
- Файл не существует — предложить создать
- Некорректные данные в Excel — пропустить/показать ошибку
- При split: предупреждение если текущий файл не сохранён
